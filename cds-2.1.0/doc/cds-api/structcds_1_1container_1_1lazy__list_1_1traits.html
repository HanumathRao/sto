Detailed Description

LazyList traits.

Either compare or less or both must be specified.

Member Typedef Documentation

typedef opt::none cds::container::lazy_list::traits::compare

Key comparing functor.

No default functor is provided. If the option is not specified, the less is used.

typedef opt::none cds::container::lazy_list::traits::equal_to

Specifies binary functor used for comparing keys for equality.

No default functor is provided. If equal_to option is not spcified, compare is used, if compare is not specified, less is used.

typedef opt::none cds::container::lazy_list::traits::less

Specifies binary predicate used for key comparing.

Default is std::less<T>.

typedef cds::sync::spin cds::container::lazy_list::traits::lock_type

Lock type used to lock modifying items.

Default is cds::sync::spin

typedef opt::v::relaxed_ordering cds::container::lazy_list::traits::memory_model

C++ memory ordering model.

Can be opt::v::relaxed_ordering (relaxed memory model, the default) or opt::v::sequential_consistent (sequentially consisnent memory model).

Field Documentation

const bool cds::container::lazy_list::traits::sort = true
static

Specifies list ordering policy.

If sort is true, than list maintains items in sorted order, otherwise items are unordered. Default is true. Note that if sort is false then lookup operations scan entire list.
